Pick #4 compile-bounded env parity

Date
2026-06-06
Topic
pick4-compile-parity
Target
orbit_wars-integration worktree (Phase 2 env-parity)

Summary

Split pick #4 into two integration slices: 4a lands pure JAX planet generation with mechanical validity and a ≤5 minute first-compile proof on the benchmark smoke path; 4b adds comets only after 4a admits. Interim integration training may run without comet physics, explicitly labeled in the cherry-pick manifest so admission results are not misread.

Problem Frame

Phase 2 pick #4 on the integration worktree has failed twice on compile economics, not parity alone. Greenfield pure JAX planet/comet generation at 75a7cf2 passed fast gates but had mechanical validity defects. A follow-up mechanical fidelity fix at 0eb349e passed 51 parity tests and trace hygiene yet caused 10+ minute JAX first-compile on the smoke/benchmark path — unacceptable for the integration loop. Integration HEAD rolled back to 9db50f5 (picks #1–2 + 3b only).

Root cause is a stacked trace: planet two-phase generation at reset plus comet spawn subgraph inlined or invoked from every step() inside the vmapped rollout trace. Parity pytest does not bound training-loop compile; a separate compile-time reject track is required.

Key Decisions

  • Two-phase pick #4 (4a planets, 4b comets). Defer comets until planet-only compile economics are proven — avoids repeating the big-bang 0eb349e pattern.
  • ≤5 minute compile ceiling. First-compile on ow benchmark training --updates 3 --warmup 1 must stay at or under five minutes for a pick to admit; 10+ minutes is a hard reject (compile_time_regression).
  • Labeled interim training. Integration may train without comets between 4a and 4b, but manifest must flag planets-only parity so operators do not treat partial physics as full mechanical fidelity.
  • Mechanical fidelity, not seed replay. JAX must emit valid rules-compliant states; maps and comet paths may differ from kaggle_environments per seed.
  • Pick #4b comets use reset-precompute. When comets land, precompute all waves at reset; step() only activates precomputed slots — no inline path search in the step trace.
  • Pick #5 blocked until 4b. Remaining mechanics hunks stay deferred until full planet+comet parity stack is green on compile + parity.

Key Flows

F1.
Pick 4a admit path
  1. Agent lands planet-only pure JAX generation on integration (no comet subsystem in env hot path).
  2. Run make test-kaggle-parity on integration — planet tests green; comet parity tests explicitly deferred/skipped with documented rationale.
  3. Run make test-jax-trace-hygiene from main harness against integration tree.
  4. Run compile smoke: ow benchmark training --preset primary --updates 3 --warmup 1 --label pick4a-compile-check.
  5. If compile ≤5 min and gates green → manifest admit pick_4a_planets; set env_parity_mode: planets_only (or equivalent manifest flag).
  6. If compile >5 min → reject compile_time_regression; rollback integration HEAD to last good SHA.
F2.
Pick 4b admit path (after 4a)
  1. Agent adds comet generation with reset-only precompute; step activates precomputed waves only.
  2. Full comet mechanical invariants green (spawn sync, initial_planets through expire, reserved comet IDs).
  3. Repeat F1 gates including full comet parity tests; compile smoke must stay ≤5 min.
  4. Manifest admit pick_4b_comets; clear planets-only flag; pick #5 unblocked for planning.
F3.
Compile cliff rollback
  1. Detect 10m+ or operator-rejected compile on smoke path.
  2. git reset --hard <last_good_sha> on integration worktree.
  3. Update manifest: verdict: reject, reject_reasons: [compile_time_regression], preserve rejected SHAs in notes.

Requirements

Compile economics

ID Requirement
R1. Every pick #4 slice (4a and 4b) must pass a compile smoke on the operator benchmark path (ow benchmark training --preset primary --updates 3 --warmup 1) with first-compile wall time ≤5 minutes before manifest admit.
R2. Compile cost is an independent reject track: green make test-kaggle-parity and trace hygiene do not admit a pick when compile exceeds tolerance. Reject reason: compile_time_regression.
R3. On compile reject, integration HEAD rolls back to last good SHA; rejected commit SHAs are preserved in manifest notes for forensics.

Pick 4a — planets only

ID Requirement
R4. 4a lands pure JAX planet generation at reset with mechanical validity: 5–10 planet groups (20–40 active planets), four-planet symmetry, at least one orbiting group, no collisions, all planets in bounds.
R5. 4a does not wire comet spawn, comet movement, or comet lifecycle into the env hot path. Comet-related parity tests are explicitly deferred — not silently skipped.
R6. Manifest records 4a as a distinct candidate (e.g. pick_4a_planets) with an explicit planets-only parity label so interim training and gate reads are not confused with full mechanical fidelity.

Pick 4b — comets (after 4a)

ID Requirement
R7. 4b precomputes all comet waves at reset; step() only activates precomputed groups on spawn steps — no inline path-search subgraph in the per-step trace.
R8. Comet mechanical invariants: planets.active == initial_planets.active through spawn and post-move expire (step 50+, step 200+); comet IDs from reserved tail slots; fourfold symmetry; valid bounds and collision rules.
R9. 4b clears the planets-only manifest flag and unblocks pick #5 for subsequent integration work.

Hot-path invariants (both slices)

ID Requirement
R10. JAX-only hot path: no pure_callback, io_callback, _reference_*, or src/game/* imports in reset/step/rollout jit paths (tier-A trace hygiene).
R11. No sequential lax.scan in _launch_fleets (pick #3 full bundle remains rejected).
R12. Per-pick fast gates pass before compile smoke: make test-kaggle-parity (integration cwd) and make test-jax-trace-hygiene (main harness).

Success Criteria

  • Pick 4a admits on integration with compile ≤5 min and planet validity invariants green — proves reset trace economics before comets.
  • Pick 4b admits with full comet mechanical invariants and compile still ≤5 min — no repeat of 0eb349e cliff.
  • Manifest truthfully represents parity mode at every HEAD (planets-only vs full).
  • Operator can re-run make gate-admission after 4b without compile blocking the milestone.

  • Sequential lax.scan fleet launch (pick #3 throughput regression).
  • Admitting picks on parity alone when compile smoke fails.

Dependencies & Assumptions

  • Integration worktree at 9db50f5 (picks #1–2 + 3b) is the rollback substrate and 4a base.
  • Admission was verified on picks #1–2; not re-run after 3b or pick #4 attempts — operator discretion for re-run timing.
  • Compile smoke uses same machine/GPU context as operator iteration for comparable wall times.
  • Reference rules in src/game/planet_generation.py and src/game/comet_generation.py anchor tests only — not hot-path imports.
  • Assumption: planet-only reset trace is the dominant compile contributor separable from comet spawn; if 4a still exceeds 5 min, bisect planet generation hunks before any comet work.
