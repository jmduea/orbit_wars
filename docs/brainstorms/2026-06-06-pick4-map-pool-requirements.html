Summary

Re-attempt pick #4 using a 500-map baked pool (planet layout + comet schedules per entry). Training-path JAX reset() indexes into the pool with round-robin rotation across reseeds — no inline generation in the JIT trace. Default pool committed in-repo; ow supports rebake and Hydra override. Eval and replay stay on kaggle_environments observation replay. A profiling gate must measure single-map bake time (with comets) before committing to the full 500-batch bake.

Problem Frame

Inline pure JAX planet generation in reset() passed parity and trace hygiene but failed the compile gate: cold-cache benchmark smoke hung 14+ minutes (same cliff class as 0eb349e). Deferring comets did not help — the rejection-sampling while_loop in reset is sufficient to brick training-loop compile. Live generation inside the vmapped hot path is ruled out for integration.

Key Decisions

  • No inline live generation in training hot path. Proven path to compile_time_regression; pool gather replaces generation in trace.
  • All training-path JAX resets use the pool — rollout, benchmark smoke, parity tests. Not eval/tournament reset.
  • Eval + replay carve-out. Submit-valid and replay paths must use kaggle_environments / observation replay — never pool reset.
  • 500 maps, round-robin rotation across reseed scheduler — reduces memorization vs fixed hash per episode.
  • Full v1 pool entries include planet layout and all comet wave schedules (not planets-only deferral).
  • Hybrid artifact stewardship. Default pool committed for reproducibility; CLI rebake and Hydra path override for experiments.
  • Profile-before-batch gate. Measure wall time to bake one full map (planets + comets) before authorizing 500-map production bake.
  • Compile admit unchanged. ≤5 minute first-compile on cold-cache benchmark smoke remains hard gate.

Requirements

Profiling & bake gates

ID Requirement
R1. Before production 500-map bake, operator/agent runs a single-map profile: wall time and success rate to generate one valid entry (planets + all comet waves) via offline reference generators. Record result in manifest or bake log.
R2. Full 500-map batch bake proceeds only after R1 profile is reviewed — extrapolated batch time (500 × per-map) must be acceptable for operator workflow (no fixed second threshold in this doc; operator judges).
R3. Every baked entry passes mechanical validity at bake time: 5–10 planet groups, symmetry, bounds, no collisions, orbiting group; comet schedules valid per reference rules (spawn steps, path bounds, fourfold symmetry).

Map pool & reset behavior

ID Requirement
R4. Pool contains exactly 500 entries unless Hydra override explicitly sets a different size for experiments (default committed artifact = 500).
R5. Training-path reset() selects map via round-robin rotation tied to reseed / episode indexing — not a static hash-only mapping for all training.
R6. Hot-path reset is gather/copy from precomputed tensors + home assignment + angular velocity — no while_loop rejection sampling, no comet path search in step().
R7. step() activates precomputed comet data from the selected pool entry only (spawn/expire/advance from baked schedules).

Artifact stewardship

ID Requirement
R8. Default map pool artifact is committed in-repo (versioned, reproducible). Manifest and training config record pool version or content hash.
R9. ow exposes rebake and optional Hydra override path for experiments without replacing the committed default silently.

Eval carve-out & hot-path invariants

ID Requirement
R10. Eval, tournament, Docker packaging replay, and submit-valid paths use kaggle_environments / observation replay — not pool-based reset().
R11. JAX-only hot path: no pure_callback, no src/game/* imports in reset/step/rollout jit paths (tier-A trace hygiene). Offline bake may use reference Python generators.
R12. No sequential lax.scan in _launch_fleets (pick #3 rejection stands).

Verification & admit

ID Requirement
R13. Per-pick fast gates: make test-kaggle-parity + tier-A trace hygiene on integration src/jax/.
R14. Compile smoke: cold-cache ow benchmark training --preset primary --updates 3 --warmup 1 first-compile ≤5 minutes before manifest admit.
R15. Manifest records env_parity_mode: map_pool, pool hash/version, R1 single-map profile metrics, and compile smoke verdict.

  • Committed 500-map pool passes bake-time validity checks (R3).
  • Cold-cache compile smoke ≤5 minutes with pool gather reset (R14).
  • Parity + trace hygiene green on integration (R13, R11).
  • Eval/replay unchanged on Kaggle path (R10).
