Problem Frame

Greenfield pick #4 at 75a7cf2 and mechanical fix 0eb349e both rolled back: parity and trace hygiene passed, but stacked planet generation + comet spawn subgraph caused 10+ minute JAX compile on training smoke. Integration HEAD is 9db50f5 with a stub reset() that does not run reference rejection-sampling planet layout. The program needs mechanical fidelity without blocking the integration loop.

Key Technical Decisions

  • KTD1 — Planets-only first slice. Do not cherry-pick 75a7cf2 wholesale; extract planet generation module and reset() wiring only. Avoids comet subgraph in training trace until compile economics are proven.
  • KTD2 — Fix two-phase loop at port time. Apply corrected _generation_done from mechanical-fidelity plan (static phase must not terminate early) when landing src/jax/planet_generation.py — do not replay the 75a7cf2 defect then fix in a second commit.
  • KTD3 — Independent px/py RNG keys. Split keys before general-phase uniform draws (fixes shared-coordinate defect from adversarial review).
  • KTD4 — Compile gate on production smoke path. Measure with uv run ow benchmark training --preset primary --updates 3 --warmup 1 --label pick4a-compile-check from main harness with integration as train cwd (or --repo-root if supported). Parity pytest alone is insufficient.
  • KTD5 — Comet tests deferred via marker. Add @pytest.mark.requires_comets (or pytest.mark.skip(reason=...) with manifest note) on comet parity tests when landing 4a; document in manifest so "parity green" is not misleading.
  • KTD6 — 4b reset-precompute only. Port src/jax/comet_generation.py from 75a7cf2 but wire precompute_comet_schedule_at_reset — no _jit_spawn_comet_group + inline lax.cond in every step() (pattern that failed at 0eb349e).
  • KTD7 — Manifest on main harness. Update docs/benchmarks/cherry-pick-manifest.json on orbit_wars main after integration picks; session handoff doc optional.

High-Level Technical Design

Pick 4a — reset-only planet port

  1. Add src/jax/planet_generation.py — pure JAX port of src/game/planet_generation.py rules (reference for tests only).
  2. Replace stub geometry in src/jax/env.py reset() with generate_planet_tables(key, cfg) (name per module API).
  3. Preserve existing home-group assignment, fleet init, encode_learner_turn call sites from pick #2 / 3b.
  4. Leave TOTAL_COMETS slots reserved but inactive; no JaxCometState, no spawn steps in step().

Pick 4b — comets after compile proof

  1. Add src/jax/comet_generation.py with path precompute at reset.
  2. Extend JaxGameState with comet schedule buffers (fixed shape, padded).
  3. step(): on spawn steps, copy precomputed group into active planet slots; expire/advance use lightweight ops only.
  4. Thread initial_planets through post-move expire (KTD from prior plan).
