```bash

uv run ow runs show --run outputs/campaigns/<c>/runs/<run_id>

uv run ow train ... artifacts=hybrid_promotion   # strict promotion: docker + tournament async
```

## Test tiers

- **Default:** `make test` / `make test-fast` — serial CPU (`-m "not slow and not jax and not sweep"`).
- **Daily (parallel wall clock):** `make test-daily` — `test-fast` ∥ `test-jax`; or `make test-daily-parallel` with xdist.
- **Kaggle env parity:** `make test-kaggle-parity` — `test_jax_env_parity.py` in the jax tier (not slow).
- **JAX trace hygiene:** `make test-jax-trace-hygiene` — tier-A `rg` gate + import freeze + fast `jit`/`collect_fn` smokes (`docs/architecture/jax-trace-tiers.md`).
- **Pre-merge:** `make test-premerge` = `test-daily` + `test-slow` (`slow and not sweep`). Before release: `make test-sweep`.
- **Parallel CPU tiers:** `make test-fast-parallel` / `make test-jax-parallel` only (sets `ORBIT_WARS_PYTEST_XDIST=1`; never on slow/sweep/GPU).
- **Never:** bare `pytest -n` without the Makefile targets; never xdist on slow/sweep; `make test-full` only when user asks (~15 min WSL2).
- **Line coverage (diagnostic):** `make test-cov-fast` (HTML in `htmlcov/`) or `make test-cov-report` (`coverage.xml`); baseline `docs/benchmarks/test-coverage-baseline.json`. Supplements parity/trace/admission gates — no CI fail-under yet.

## Key invariants

- Feature path: obs → `encode_turn` (`TurnBatch`); golden tests in `tests/test_feature_encoding_golden.py`.
- Rollout metrics: merge with `_merge_metric_dicts` inside `jax.lax.scan`; finalize rates after the scan.
- Priorities: human `docs/ROADMAP.md` (no automated validation).
- **Verification thresholds:** derive pass/fail bars from measured calibration or baseline runs (`docs/benchmarks/preflight-calibration.json`, `make preflight-calibrate`); never invent round numbers or relax a threshold until a run passes — that is not verification.

<!-- preflight-thresholds -->
- **Calibrated learning signal (Gates 2–4):** `window_updates=10`, `min_win_rate_delta=0.05`, `max_approx_kl=0.15`, `min_entropy=0.0001` — source `docs/benchmarks/preflight-calibration.json` (commit `614cf36e732c`).
- **Tournament win proof (Gate 5):** `noop_min_win_rate=0.7`, `random_min_win_rate=0.58`.
<!-- /preflight-thresholds -->

- **Metric gates:** before gating on a training metric, confirm its denominator and what “chance” means for that opponent and reward mode (e.g. `overall_win_rate` vs `noop_only` is not ~50%; `episode_reward_mean` under `binary_win` is not win rate; self-play ~50% is not a learning signal).
- **Seed scheduler:** default `training.reseed_every_updates=50` (calibrated 2026-06-03, `docs/benchmarks/seed-scheduler-calibration.json`); use `-1` for auto `max(25, total_updates // 10)`. Run `ow benchmark calibrate-seed-scheduler` before changing. Reseed resets rollout env state, not just the PRNG key.

## Learned User Preferences

- Prefer unified v2-only feature encoding and full rework over shims when simplifying encoding, rollout, training, or refactors — remove legacy v1 paths; no backward-compat re-export modules, `_underscore` aliases, parallel APIs, or duplicate Hydra/W&B sweep recipes (`wandb_sweep=preflight` canonical — not `ssot_preflight`); update call sites instead. Same for opponents/curriculum: collapse dual Hydra surfaces so config knobs match runtime behavior (no `mix.weights`-only nearest/sniper profiles; planned hard cutover to curriculum-owned opponent selection).
- Daily dev loop: `make test-fast` or a domain Makefile target — not bare full `pytest` and not slow/JAX-compile smokes unless explicitly requested; use `make test-fast-parallel` / `make test-jax-parallel` for CPU xdist only — never xdist on slow/sweep or with GPU pytest.
- **Do not commit** unless the user explicitly asks; **do not push** to remote unless the user explicitly requests it.
- Check terminals / `make agent-context` `gpu_contention` before GPU work; respect `.cursor/hooks` `beforeShellExecution` denial when another repo terminal is active; never run multiple concurrent `wandb agent` processes on one GPU; pin each W&B sweep agent to one branch/commit for the whole sweep — `rollout_s` outliers between arms usually trace to git checkout drift (e.g. hygiene cherry-pick landing mid-sweep), not sampled hyperparams; tag/compare runs via manifest `git.commit`. Run ce-optimize GPU rollout profiles one at a time — parallel profiles on one GPU abort with no JSON. W&B System GPU Utilization % is a coarse NVML average — 55–65% on mixed-format rollouts is structurally plausible; optimize `rollout_s` / `rollout_env_steps_per_sec`, not 100% util, as throughput success.
- **Long `ow` commands:** never pipe to `tail`, `head`, or `2>&1 | tail` — the pipe hides progress until exit and looks hung. Gate/calibration train output streams on **stderr**; keep stdout for final JSON or use `--out path.json`. Prefer `ow benchmark gate run <id> --verbose --out /tmp/gate.json`, `ow runs watch`, `ow eval status --watch`, or `tail -f outputs/campaigns/<c>/runs/<id>/logs/*_jax.jsonl` for live metrics. Pass Hydra overrides as **separate CLI arguments** on `ow train`, not one space-joined string; on `ow benchmark gate run` put all overrides in **one** `--train-overrides` flag (space-separated inside the value — repeated `--train-overrides` last-wins).
- New train/eval/benchmark capabilities belong in the `ow` CLI (`src/cli/<module>.py` + dispatch in `src/cli/__init__.py`), not standalone `scripts/*.py`; defer heavy imports (JAX) until command execution, not module load. Benchmark layout: measurement in `src/benchmark/`, CLI dispatch in `src/cli/benchmark/`, gate recipes in `conf/benchmark/gates/`. For agent loops prefer **primitive** subcommands (`ow runs`, `ow eval status`, `ow eval jobs cancel`, `ow benchmark gate run admission`, `make gate-admission`, `ow benchmark tournament-proof`, `ow sweep`) over workflow wrappers (`learn-proof`, hybrid promotion train); `ow benchmark admission-throughput` is post-hoc extract only; surface operator commands directly in responses — task prompts in `docs/AGENT_CAPABILITIES.md`. Interactive rollout debugging: `ow debug step|game` (Tier C eager host driver, `JAX_DISABLE_JIT=1`) — not IDE breakpoints inside jitted `ow train` `collect_fn`/`lax.scan`.
- Prefer `task=shield_cheap` or `task=shield_off` for training experiments; avoid `shield_tiered` unless explicitly requested.
- User-facing summaries and docs: plain language; avoid internal jargon (tier/U1/rank labels, raw SHAs) — use informative commit/subject names in prose, manifest, and checklists; reserve SHAs for git commands only; internal names OK in manifests/agent context; see `docs/nomenclature-rfc.md` for term mappings.
- Pipeline validation (SSOT spine, new train funnels, Colab long runs): walk through stages step-by-step with the user — do not launch autonomous long E2E runs or Colab launches without explicit per-step approval. Colab v1 is compute-only after local preflight; long runs use fixed-path pilot + `--monitor-after-launch` (not map_pool+production_mix defaults); one Colab session at a time; no `preflight` W&B tag on long runs. When asked to locate an issue source without fixing ("don't touch, just point me where"), respond with file/path pointers only — no code edits.
- **Single-worktree discipline:** all feature work, sweeps, and gate prep only in `orbit_wars-integration` on `refactor/artifacts-metric-promotion-commit` — verify `git branch`/checkout before every edit; `orbit_wars` is admission harness only (`make gate-admission REPO_ROOT=../orbit_wars-integration`), no feature commits; pinned siblings are baseline refs, not daily dev; no cherry-picks/landings/rollbacks without explicit user ask. Active cherry-pick direction is **main→integration** (env-parity substrate); **integration→main** only after `admission_passed` + manifest `ready_for_main` — not today's workflow. Learning proof outranks throughput; one unified admission recipe; **incremental gating** — anchor integration, cherry-pick one SHA at a time with per-pick gates before stacking the next.
- Phase 2 env parity targets **mechanical fidelity** (valid rules-compliant JAX states), not seed/bit-exact replay parity with `kaggle_environments`; maps, comet paths, comet RNG strings, and home-group PRNG stream may differ per seed — gate on validity invariants, not coordinate goldens.
- `agent-browser` and similar automation must surface pages to the user (`open_resource` in Glass, `xdg-open`, or a visible browser) — not headless agent-only sessions; submit-valid paths require Docker/packaging validation before any `kaggle_environments` tournament ladder — do not spend tournament compute on checkpoints that would fail Kaggle upload.
